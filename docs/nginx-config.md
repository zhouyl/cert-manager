# Nginx 配置生成功能

cert-manager 支持自动生成 Nginx SSL 配置文件，方便在证书部署时同时更新 Nginx 配置。

## 功能特性

- 🔧 **自动生成配置**：证书部署时自动生成 Nginx SSL 配置文件
- 📝 **模板支持**：支持自定义配置模板
- 🔄 **变量替换**：支持域名和证书目录变量替换
- 📁 **灵活路径**：支持自定义配置文件路径
- 👀 **预览功能**：部署前可预览生成的配置内容

## 配置方法

### 1. 在服务器配置中添加 cert_conf_file

编辑 `config/config.yaml`，在服务器配置中添加 `cert_conf_file` 字段：

```yaml
servers:
  - name: "nginx-server"
    host: "192.168.1.100"
    port: 22
    username: "root"
    identity_file: "~/.ssh/id_rsa"
    cert_directory: "/etc/nginx/certs"
    cert_conf_file: "/etc/nginx/includes/{domain}conf"
    reload_command: "systemctl reload nginx"
    reload: true
```

### 2. 支持的变量

- `{domain}`：会被替换为实际的域名
- `{cert_directory}`：会被替换为证书目录路径

### 3. 配置文件模板

默认模板位于 `config/nginx-ssl.conf.template`：

```nginx
# Nginx SSL Certificate Configuration Template
# This file is automatically generated by cert-manager
# Domain: {domain}
# Generated at: {generated_at}

ssl_certificate {cert_directory}/fullchain.pem;
ssl_certificate_key {cert_directory}/privkey.pem;
# ssl_trusted_certificate {cert_directory}/fullchain.pem;
ssl_session_cache shared:SSL:10m;
ssl_session_timeout 1h;
ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:ECDHE:ECDH:AES:HIGH:!NULL:!aNULL:!MD5:!ADH:!RC4;
ssl_protocols TLSv1.2 TLSv1.3;
ssl_prefer_server_ciphers on;
```

## 使用方法

### 1. 预览配置

在部署前预览将要生成的 Nginx 配置：

```bash
# 预览所有服务器的配置
python cert_manager.py nginx preview example.com

# 预览指定服务器的配置
python cert_manager.py nginx preview example.com --server nginx-server
```

### 2. 自动生成

在证书部署时会自动生成配置文件：

```bash
# 部署证书时自动生成 Nginx 配置
python cert_manager.py deploy example.com
```

### 3. 在 Nginx 中使用

在你的 Nginx 站点配置中包含生成的配置文件：

```nginx
server {
    listen 443 ssl http2;
    server_name example.com *.example.com;
    
    # 包含自动生成的 SSL 配置
    include /etc/nginx/includes/example.com.conf;
    
    location / {
        # 你的站点配置
    }
}
```

## 配置示例

### 示例 1：基本配置

```yaml
servers:
  - name: "web-server"
    host: "192.168.1.10"
    cert_directory: "/etc/nginx/certs"
    cert_conf_file: "/etc/nginx/conf.d/ssl/{domain}conf"
```

生成的配置文件路径：`/etc/nginx/conf.d/ssl/example.com.conf`

### 示例 2：多服务器配置

```yaml
servers:
  - name: "nginx-1"
    host: "192.168.1.10"
    cert_directory: "/etc/nginx/certs"
    cert_conf_file: "/etc/nginx/includes/{domain}conf"
    
  - name: "nginx-2"
    host: "192.168.1.11"
    cert_directory: "/var/ssl/certs"
    cert_conf_file: "/etc/nginx/ssl-configs/{domain}ssl.conf"
```

### 示例 3：不生成配置

如果不需要生成 Nginx 配置，只需不设置 `cert_conf_file` 字段：

```yaml
servers:
  - name: "apache-server"
    host: "192.168.1.20"
    cert_directory: "/etc/apache2/ssl"
    # 不设置 cert_conf_file，不会生成 Nginx 配置
```

## 自定义模板

你可以修改 `config/nginx-ssl.conf.template` 文件来自定义生成的配置内容。

支持的模板变量：
- `{domain}`：域名
- `{cert_directory}`：证书目录
- `{generated_at}`：生成时间

## 注意事项

1. **权限**：确保 SSH 用户有权限在指定路径创建文件
2. **目录**：配置文件目录会自动创建
3. **权限设置**：生成的配置文件权限为 644
4. **备份**：建议在修改模板前备份原文件
